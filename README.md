# 🚀 Git шпаргалка (быстрый ПУШ)

## 📤 Отправить изменения на GitHub

### Полный вариант

```bash

git add README.md

git commit -m "описание изменений"

git push origin master

-------------------------------------
**Пояснения.**

git add .	                    - Добавить все изменённые файлы

git add README.md	            - Добавить только README

git commit -m "текст описания"	- Сохранить изменения с описанием

git push origin master	        - Отправить на GitHub

-------------------------------------

**Мой рабочий цикл**

0. Внести изменения в файлы, тесты, readme.

1. Изменить файл → Ctrl + S

2. git add .

3. git commit -m "что сделал"

4. git push

-------------------------------------

```

# 🚀 Git шпаргалка (быстрый ПУЛ)

## 📤 Забрать изменения с GitHub

### Полный вариант

```bash

git pull origin master

### Быстрый вариант (если ветка master уже отслеживается)

git pull

-------------------------------------
**Пояснения.**

git pull origin master - Забрать изменения с GitHub в ветку master
git pull	             - Забрать изменения с текущей ветки (коротко)
git fetch	             - Посмотреть, есть ли изменения (без загрузки)

-------------------------------------

**Мой рабочий цикл (когда менял README на GitHub)**

0. Изменил README на сайте GitHub (или коллеги залили код)

1. git pull

2. Файлы обновились локально

3. Продолжаю работать

-------------------------------------

```
